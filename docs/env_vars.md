# Environment Variables

This document is the source of truth for Printy deployment configuration.

Production targets:
- Frontend: `https://printy.ke`
- API: `https://api.printy.ke`
- Canonical M-Pesa callback: `https://api.printy.ke/api/payments/mpesa/callback/`

Templates in this repo:
- `printy_api/.env.example` is the production-safe template.
- `printy_api/.env.local.example` is the localhost development template.
- `printy_ui/.env.example` is the production-safe frontend template.
- `printy_ui/.env.local.example` is the localhost frontend template.

## Backend required for production

| Variable | Example | Notes |
| --- | --- | --- |
| `SECRET_KEY` | `replace-with-long-random-secret` | Required. Never commit. |
| `DEBUG` | `false` | Must be `false` in production. |
| `ALLOWED_HOSTS` | `api.printy.ke,printy.ke,www.printy.ke` | Add IP only if operationally needed. |
| `SITE_DOMAIN` | `printy.ke` | Run `python manage.py configure_site` after changes. |
| `SITE_NAME` | `Printy` | Used by Django Sites and auth flows. |
| `FRONTEND_URL` | `https://printy.ke` | Required for email links. Backend fails if unsafe when `DEBUG=False`. |
| `CORS_ALLOWED_ORIGINS` | `https://printy.ke,https://www.printy.ke` | Add local dev origins only in local env. |
| `CSRF_TRUSTED_ORIGINS` | `https://printy.ke,https://www.printy.ke,https://api.printy.ke` | Required for admin/allauth form origins. |

| Variable | Example | Notes |
| --- | --- | --- |
| `MPESA_ENV` | `production` | `sandbox` for local/testing only. |

| `MPESA_CONSUMER_KEY` | `replace-with-prod-consumer-key` | Must match the same Daraja environment as the shortcode/passkey. |
| `MPESA_CONSUMER_SECRET` | `replace-with-prod-consumer-secret` | |
| `MPESA_SHORTCODE` | `replace-with-prod-shortcode` | |
| `MPESA_PASSKEY` | `replace-with-prod-passkey` | |
| `MPESA_CALLBACK_URL` | `https://api.printy.ke/api/payments/mpesa/callback/` | Canonical callback. Backend fails if unsafe in production. |
| `MPESA_TIMEOUT_SECONDS` | `30` | |
| `MPESA_ACCOUNT_REFERENCE_DEFAULT` | `PRINTY` | |
| `MPESA_TRANSACTION_DESC_DEFAULT` | `Printy payment` | |
| `MPESA_INITIATOR_NAME` | optional | Future B2C/status flows only. |
| `MPESA_INITIATOR_PASSWORD` | optional | Future B2C/status flows only. |
| `MPESA_SECURITY_CREDENTIAL` | optional | Future B2C/status flows only. |
| `MPESA_TIMEOUT_URL` | optional | Future timeout/result flows only. |
| `MPESA_RESULT_URL` | optional | Future timeout/result flows only. |

Production safety behavior already enforced in code:
- `SECRET_KEY` cannot stay as a placeholder when `DEBUG=False`
- `ALLOWED_HOSTS` must include non-local production hosts when `DEBUG=False`
- `CSRF_TRUSTED_ORIGINS` must be HTTPS when `DEBUG=False`
- `CORS_ALLOWED_ORIGINS` must be HTTPS when `DEBUG=False`
- `FRONTEND_URL` cannot be localhost when `DEBUG=False`
- `FRONTEND_URL` must be HTTPS when `DEBUG=False`
- `MPESA_CALLBACK_URL` must be set, public, and HTTPS when `MPESA_ENV=production`
- `MPESA_CALLBACK_URL` cannot be localhost when `MPESA_ENV=production`

## Frontend required for production

| Variable | Example | Notes |
| --- | --- | --- |
| `NUXT_PUBLIC_API_BASE_URL` | `https://api.printy.ke/api` | Required for the deployed frontend. |
| `NUXT_PUBLIC_SITE_URL` | `https://printy.ke` | Used for canonical runtime site references. |

Production frontend safety behavior:
- the rebuilt frontend defaults to `https://api.printy.ke/api` in production
- the frontend throws if `NUXT_PUBLIC_API_BASE_URL` is invalid or points to localhost in production

## Local development differences

Local dev may use:
- `DEBUG=true`
- `FRONTEND_URL=http://localhost:3000`
- `NUXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000/api`
- local CORS and CSRF origins for `localhost:3000`, `127.0.0.1:3000`, `localhost:5173`
- `MPESA_ENV=sandbox`
- a public tunnel callback URL if you need real sandbox callbacks to reach your local machine

Production examples in this repo should never use localhost as the intended deployment value.
